# Step 9: Workflow - Submitting a PR

This is the most important moment. You are officially asking to merge your work into the main project!

## 📤 1. Open the Pull Request
Go to the original project on GitHub. You should see a yellow banner saying **"Compare & pull request"**. Click it!

## 📝 2. Describe Your Changes
A good Pull Request (PR) description makes it easy for maintainers to say "Yes".
- **Title**: Brief and descriptive (e.g., `docs: fix typo in Step 4`)
- **Body**: Use a template like this:
  > **What does this PR do?**
  > Fixes a spelling error in the Git fundamentals section.
  > 
  > **Which issue does it relate to?**
  > Fixes #123

## 💎 3. The "Polished" PR
- Add a screenshot if you changed something visual.
- Check for typos in your own description.
- Be ready for feedback (Maintainers might ask for small changes).
