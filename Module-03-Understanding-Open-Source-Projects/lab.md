# Module 3 Lab: Deconstructing a Repository

In this lab, you will act as a researcher. You will find a popular project and dissect its documentation.

## 🏗️ Objectives
1.  Locate standard files (`CONTRIBUTING`, `LICENSE`).
2.  Analyze active issues.
3.  Evaluate the project's welcoming atmosphere.

## 🛠️ Step-by-Step Instructions

### Step 1: Pick a Project
Choose a well-known project on GitHub. Examples:
- [React](https://github.com/facebook/react)
- [Python Requests](https://github.com/psf/requests)
- [VS Code](https://github.com/microsoft/vscode)

### Step 2: Find the "Rules"
- Can you find the `CONTRIBUTING.md`? Open it.
- Does it have a specific step for setting up the environment?
- Look for the `LICENSE`. What type is it? (MIT, Apache, etc.)

### Step 3: Analyze the Issues
- Go to the **Issues** tab.
- Search for the label `good first issue`.
- Pick one issue and read the comments. Does the maintainer seem helpful? Is the issue clearly explained?

### Step 4: The "Readme" Test
- Read the first paragraph of the `README.md`.
- In your own words, what does this project do?
- Is it clear how to install it?

---

## ✅ Deliverable
Write a short summary (100 words) of the repository you analyzed. include the project name, the license type, and whether you found any "good first issues".
