# Module 3 Self-Test: Understanding Open Source Projects

Test your knowledge of repository navigation and etiquette.

## ❓ Questions

1.  **Which file should you read to learn the rules for submitting code?**
    - A) `README.md`
    - B) `LICENSE`
    - C) `CONTRIBUTING.md`
    - D) `TODO.txt`

2.  **What does the label "Good First Issue" usually signify?**
    - A) The issue is for senior developers only.
    - B) The issue is beginner-friendly and requires little project context.
    - C) The issue has already been resolved.
    - D) The issue is a legal requirement.

3.  **Which license is generally considered "permissive," allowing you to use it for almost anything?**
    - A) GPL
    - B) MIT
    - C) Proprietary
    - D) Creative Commons BY-ND

4.  **How should you "claim" an issue on GitHub?**
    - A) Just start writing code and hope for the best.
    - B) Email the CEO of the company that owns the project.
    - C) Comment on the issue expressing interest and asking to be assigned.
    - D) Delete the issue so no one else sees it.

5.  **True or False: The `README.md` is usually the first file a user or contributor sees.**

---

## 🔑 Answer Key
1. C
2. B
3. B
4. C
5. True
