# Step 5: Navigating Open Source Projects

Every GitHub repository (Project) has a specific "Anatomy". Learning to read it is your first step to being a great contributor.

## 📁 The "Big Three" Files
1.  **README.md**: The "Door". Read this first to understand what the project does.
2.  **LICENSE**: The "Rules". This tells you how you can use the code.
3.  **CONTRIBUTING.md**: The "Guidebook". This contains the specific steps for submitting changes to *this* project. **Always read this before starting work.**

## 🔍 The Infrastructure
- **Issues**: Where bugs and feature requests live. Look for labels like `good first issue` or `beginner-friendly`.
- **Pull Requests**: Where you can see what others are working on.
- **Discussions/Wiki**: Where communities plan and document complex ideas.

## 💡 Pro Tip
Check the "Last Update" date. If the project hasn't been touched in 2 years, it might be "unmaintained," and your PR might not be reviewed.
