# Step 6: Finding Issues and Etiquette

Now that you know how projects are built, it's time to find your first task.

## 🎯 How to Find "Good" Issues
Don't try to fix the core engine on your first day. Look for these labels in the "Issues" tab:
- `good first issue`
- `documentation`
- `beginner-friendly`
- `help wanted`

## 💬 Community Etiquette
Before you start coding, COMMUNICATE.
- **Ask Permission**: Comment on the issue: *"Hi, I'd like to work on this. Can you assign it to me?"*
- **Be Polite**: Remember maintainers are often doing this for free.
- **Don't Ghost**: If you realize you can't finish the task, let them know so someone else can take it.

## 🤝 The "Success" Pattern
1. Find an issue labeled `documentation`.
2. Ask if it's still available.
3. Once assigned, move to Step 7.
