# Module 4 Self-Test: The First Contribution

Final check before you head out into the open-source community!

## ❓ Questions

1.  **What is a "Fork" on GitHub?**
    - A) A tool used for fixing hardware bugs.
    - B) A copy of a project that is added to your own account.
    - C) A way to delete a project from the internet.
    - D) A private branch that only maintainers can see.

2.  **Why should you create a feature branch instead of working on `main`?**
    - A) `main` is protected and you can't push to it in most projects.
    - B) It keeps your changes organized and separate from the official code.
    - C) It makes it easier for you to work on multiple bug fixes at once.
    - D) All of the above.

3.  **What does "Pushing" code do?**
    - A) Deletes the code from your computer.
    - B) Sends your local commits to the remote repository (GitHub).
    - C) Asks the maintainer to merge your code immediately.
    - D) Renames all your variables to "Push".

4.  **What should you do if a maintainer asks for changes in your Pull Request?**
    - A) Delete the PR and never talk to them again.
    - B) Argue that your code is perfect.
    - C) Make the suggested changes in your local branch, commit, and push.
    - D) Open a new PR with the changes.

5.  **Which of these is a good practice for PR descriptions?**
    - A) Leaving it blank to save time.
    - B) Explaining *what* changed and *why*, and linking the issue number.
    - C) Writing a 10-page essay on the history of software.
    - D) Asking the maintainer for a job.

---

## 🔑 Answer Key
1. B
2. D
3. B
4. C
5. B
