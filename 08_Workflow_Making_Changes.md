# Step 8: Workflow - Making Changes

This is where you actually do the work. Remember: simple is better.

## ✍️ 1. Make Your Edits
Open the project in VS Code (or your chosen editor) and make the changes you discussed in the issue.
- If it's a typo, fix the spelling.
- If it's code, ensure it follows the project's style.

## 🧪 2. Test Your Work
Don't break things!
- If the project has tests, run them (check `README` or `CONTRIBUTING` for how).
- If it's documentation, read through it to ensure it looks correct.

## 📸 3. Stage and Commit
Once you are happy, save your work in Git:
```bash
git add .
git commit -m "docs: fix spelling error in first-contribution guide"
```

## ☁️ 4. Push to Your Fork
Send your local branch up to GitHub:
```bash
git push origin fix/typo-in-guide
```
