# Contributing to the Roadmap

First off, thank you for considering contributing to the **Open Source Contribution Roadmap**! It’s people like you that make open source such a fantastic environment for learning.

## 🌈 How You Can Help
You don't need to be an expert to contribute. You can:
- **Fix Typos**: Spotted a spelling error? Submit a PR.
- **Improved Explanations**: If a concept was hard to understand, suggest a clearer way to write it.
- **Add Resources**: Found a great tool or tutorial? Add it to `resources/learning-resources.md`.
- **New Modules**: Want to add a "Module 5: Advanced Git"? We'd love that!

## 🛠️ Contribution Workflow
1.  **Fork** this repository.
2.  **Clone** your fork locally.
3.  **Create a branch** for your changes (`git checkout -b fix/typo-module-1`).
4.  **Make your changes** and commit them with a descriptive message.
5.  **Push** to your fork.
6.  **Open a Pull Request** against our `main` branch.

## 📝 Code Style
- Use clear, beginner-friendly English.
- Use emojis sparingly to highlight important sections.
- Ensure all markdown links are working.
