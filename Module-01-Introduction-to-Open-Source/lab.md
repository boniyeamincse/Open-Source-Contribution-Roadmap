# Module 1 Lab: Exploring GitHub

In this lab, you will get hands-on by exploring the world's largest host of open-source projects: **GitHub**.

## 🏗️ Objectives
1.  Search for projects that interest you.
2.  Understand how to navigate a repository.
3.  Find "Good First Issues".

## 🛠️ Step-by-Step Instructions

### Step 1: Create a GitHub Account
If you don't have one, head over to [GitHub.com](https://github.com/join) and sign up. It's free!

### Step 2: Explore Top Repositories
- Go to the [GitHub Explore](https://github.com/explore) page.
- Look at "Trending" repositories to see what people are building right now.

### Step 3: Use the Search Feature
- Think of a technology you like (e.g., "Python", "React", "Minecraft").
- Search for it in the GitHub search bar.
- Filter by "Repositories" and sort by "Most Stars" to see the most popular ones.

### Step 4: Find Your First Issue
- Use this special URL to find issues that are beginner-friendly:
  [https://github.com/issues?q=is:open+is:issue+label:"good+first+issue"](https://github.com/issues?q=is:open+is:issue+label:%22good+first+issue%22)
- Browse through them and see if any seem interesting (even if you don't understand the code yet).

---

## ✅ Deliverable
Write down the names of **three** open-source projects you find interesting and one "Good First Issue" that you might want to look into later.
